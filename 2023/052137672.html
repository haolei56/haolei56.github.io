inline

inline修饰函数，是给编译器的一个建议，到底是否为inline还得由编译器来决定，关键字inline必须与函数定义放在一起，inline修饰的函数在使用时是做简单的替换，有点类似于宏定义。这样就避免了一些函数栈空间的使用，从能提升效率。inline推出的目的，也正是为了取代c语言中的宏定义表达式。

extern

1、作用在变量之前：变量只允许定义一次，但可以在多个文件中声明。

Test.cpp 中：

int s32Val = 0;     // 定义一个变量 s32Val，并赋初值为 0

Test1.cpp 中：

extern int s32Val;  // 声明变量 s32Val，它在 Test.cpp 中被定义，此处不可赋值

2、作用在函数之前：用于标识函数为外部可调用函数

Test.h：

extern void Fun();   // 函数声明，extern 用于标识此函数为外部可调用函数

Test.cpp：

void Fun();  // 函数定义

const

  1. 加在参数列表里

    void function（const int& tmp）{}//tmp变量在函数体内不能被更改

  2. 加在类中某个函数名后面

    void function const（）{}//类的成员变量在函数体内不能被更改，不改变成员变量的函数一定要加。不希望且不会改变类的成员变量时一律加上const关键字

  3. 加在类对象前

    const class_name& tmp；// tmp对象的里面的值不能修改，能够调用2中的常量成员函数。类成员函数没有声明成cosnt，这样就会报错，因为其可能修改对象里的值。

三大构造函数：拷贝构造、拷贝赋值、析构

构造函数，尽可能使用初值列的形式给成员变量赋值；

以带指针成员的String类为例

String(const String& str); // 函数接受的是自己类型的东西，称之为拷贝构造函数。
String& operator=(const String& str); // 拷贝赋值函数。
~String(); // 析构函数

String s1("hello");
String s2(s1);//调用拷贝构造函数
String s2=s1;//调用拷贝赋值函数

浅拷贝：把一个指针变量赋值给另一个指针变量，b指针直接指向a指针，造成b指针原来指向的空间没有指针指向，形成了内存泄露，很危险。

  • 拷贝构造

    String::String(const String &str) {
    m_data = new char[strlen(str.m_data) + 1];//创建空间
    strcpy(m_data, str.m_data);//再拷贝
}

    深拷贝：先创建足够空间，然后把被拷贝的值复制过来。

  • 拷贝赋值

    String::String(const String &str) {
    if (this == &str) {//检测自我赋值
        return *this;
    }
    delete[] m_data;
    m_data = new char[strlen(str.m_data) + 1];
    strcpy(m_data, str.m_data);
    return *this;
}

    1. 先清空原来的数据，delete掉自己；

    2. 然后重新分配空间。

    3. 拷贝过来

      需要注意，这里检测自我赋值很重要，不可忽略。否则第一步会把自己给删除了，从而报错。

区别：

  1. 拷贝构造函数是一个对象初始化一块内存区域，这块内存就是新对象的内存区，而赋值构造函数是对于一个已经被初始化的对象来进行赋值操作。
  2. 拷贝构造函数首先是一个构造函数，它调用时候是通过参数的对象初始化产生一个新对象。赋值构造函数是把一个新的对象赋值给一个原有的对象。

对于不带有指针的类，这3个函数可以使用编译器默认生成的版本；但是编写带有指针的类时就必须定义这3个特殊函数。

constc

值传递和引用传递

无论是函数参数还是函数返回类型，建议使用引用传递，尽量不要值传递。引用传递底部是一个指针，一个指针四个字节，很快。传引用就相当于传指针，所以引用一改全改，如果希望传进来的参数不被改变，则要在参数前加const。

函数返回值也尽量使用引用。在函数内部新建的临时变量不能做为引用返回，如果返回引用，这个新对象在出了函数体之后，生命周期就结束了，会出现错误，此时只能返回值。

友元

相同class的各个对象(objects)互为友元(friends)

class complex
{
public:
    complex (double r = 0; double i = 0): re (r), im (i){ }

    int func(const complex& param){ 
        return param.re + param.im; //这个怎么可以直接拿
    }

private:
    double re, im;
};

{
    complex c1(2,1);
    complex c2;
    c2.func(c1);
}

内存管理

C++内存模型分成五个部分：堆、栈、全局/静态存储区（C语言还分.bss段和.data段）、常量区、代码区

  • **Stack:**是存在于某作用域(scope)的一块内存空间(memory space)。例如当你调用函数，函数本身即会形成一个stack用来放置它所接收的参数，以及返回地址。在函数体内声明的任何变量，其所使用的内存块都取自上述stack。

  • **Heap:**或谓system heap，是指由操作系统提供的一块global内存空间，程序可动态分配(dynamic allocated)从某中获得若干区块(blocks)。

    class Complex{...};
...
Complex c3(1, 2);
...
{
    Complex c1(1, 2);//在栈里
    static Complex c2(1, 2);
    Complex* p = new Complex(3);//在堆里
    ...
    delete p;
}

    c1便是所谓stack object，其生命在作用域{ }(scope)结束之后结束。这种作用域内的object，又称为auto object，因为它会被“自动”清理，自动调用析构函数。

    c2便是所谓的static object，其生命在作用域(scope)结束之后仍然存在，直到整个程序结束。

    c3便是所谓global object，其生命在整个程序结束之后才结束。也可以把它视为一种static object，其作用域是“整个程序”。

    p所指的便是heap object，其生命在它被deleted之后结束。如果没有delete，会造成内存泄漏(memory leak)，因为当作用域结束，p所指的heap object仍然存在，但指针p的生命却结束了，作用域之外再也看不到p（也就没机会delete p）。

  • new的内部步骤：1.分配内存 2.转化类型 3.调用构造函数

    Complex* pc = new Complex(1, 2);

//编译器转化为如下步骤
Complex *pc;
void* mem = operator new(sizeof(Complex));//分配内存，operator new是一个函数，其内部调用malloc(n)
pc = static_cast(mem);//转型
pc->Complex::Complex(1, 2);//构造函数

  • delete步骤：1.调用析构函数 2.释放内存。析构函数先删除内容，delete删除指针

    Complex* ps = new Complex(1, 2);
...
delete ps;

//编译器转化为如下步骤
Complex::~Complex(ps);//析构函数
operator delete(ps);//释放内存，delete函数内部调用free(ps)

    注意：new数组一定要搭配数组delete(delete [] )

static

静态函数和一般成员函数的区别：静态函数没有this pointer。静态函数只能处理静态数据

类的静态成员变量需要在类外部进行定义。

调用static函数的方式有二：
（1）通过object调用
（2）通过class name调用

组合(复合)、继承、委托

组合：has-a，一个类中的成员变量含有另一个类的对象，图表中的表现是实心菱形
继承：is-a
委托：有一个另一个类的指针，composition by reference，图表中的表现是空心菱形(只有一个 指针，比较虚)

组合的构造和析构：

  1. 构造，由内而外，先构造成员类，后构造自己
  2. 析构，由外而内，先析构自己，够析构成员类

继承关系下的构造和析构：

  1. 构造，由内而外，先构造父类，后构造自己
  2. 析构，由外而内，先析构自己，够析构父类
  3. 存在继承关系时，父类的析构函数必须要设计为虚函数

继承+复合(子类继承父类，子类里又有组合)关系下的构造和析构：

  1. 构造，先调用父类的构造，然后调用复合的构造，最后调用自己的构造
  2. 析构，先调用自己的析构，然后调用复合的析构，最后调用父类的析构

虚指针&虚函数表

一个类占的内存大小包括:成员变量，父类的成员变量，如果有虚函数则有一根虚函数表指针

普通函数：不希望子类重新定义(override,复写)它。
虚函数：希望子类重新定义(override，复写)它，子类也可以不复写它，它已有默认定义。
纯虚(pure virtual)函数：希望子类一定要重新定义(override)它，对它没有默认定义。

继承了一个有虚函数表的类，那么子类也会有自己独立的虚函数表，如果没有重写，那么这个独立的虚函数表指向的函数地址不变，但是如果重写了，就会将对应的函数地址指针指向改变，就相当于有一个自己的实现，这才能完成多态。A *p = new B()，B继承A，B有自己的虚函数表，A类型的指针指向了B类的地址，使用A调用虚函数的时候，其实是调用B类的虚函数表中的函数，所以叫之为动态绑定，但是使用A调用非虚函数的话，还是使用的是A自己里面的非虚函数，本质上来说，指针p还是A类型的，而且A类型里面的非虚函数没有经过动态编译。通过对象调用，不会动态绑定，只有通过指针调用才会动态绑定。

pointer-like，智能指针

比如说shared_ptr重载了运算符*和->，迭代器也可以看作是一种智能指针

function-like，仿函数

重载()

操作符重载

在C++中的操作符重载有两种形式，一种是在类中作为成员函数的操作符重载，另一种是在类外作为全局函数的操作符重载。

1、在类中作为成员函数的操作符重载，这种情况下操作符重载参数中隐含有this指针，调用时操作符作用在左操作数上；操作符重载与函数重载不同，函数重载在调用时仍然是obj.func()的形式，而操作符重载使用时， 例如operator += (const complex&)可以认为满足这种形式this += const complex&就要调用这个函数。并不需要.()这种调用形式。

2、在类外作为全局函数的操作符重载，无this指针。complex operator + (const complex& x, const complex& y)，相当于把+放在参数之间，即遇到这种形式complex& + complex& 就要调用该函数。
