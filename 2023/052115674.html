C++标准库

C++标准库（C++ Standard Library）：通过include头文件调用，标准库中70%都是STL(Standrad Template Library)；

C++标准库.h文件，编译器自带，可以看到源代码。与C不同，C中的头文件一般写法带有.h，例如#include<stdlib.h>，C++中去掉.h，在前面加字母C，即#include<cstdlib>。新式头文件基本都封装在std名称空间中，using namespace std;

STL六大部件：
（1）容器(Container);（2）分配器(Allocators)；（3）算法(Algorithms)；
（4）迭代器(Iterators)；（5）适配器(Adapters)；（6）仿函式(Functors)

分配器负责给容器分配内存，适配器这可以把容器类型进行转换。

注意：迭代器属于前闭后开区间，begin()指向开始位置，end()指向结束的下一位置。

容器

大致分为两种容器：序列容器，关联容器
所谓关联容器，就是key和value的结构
序列容器：array、vector、deque、list、forward-list
关联容器：set、multiset、map、multimap，实现使用红黑树（高度平衡二叉树）做的
C++11中有Unordered容器，但他也属于关联容器，实现使用hash table做的

Array

C语言中就有数组，C++把它包装成容器，让它有容器的行为。必须指定大小，不能拓展。没有构造函数，没有析构函数！

array c;
c.size();
c.front();//返回array的第一个元素
c.back();//返回array的最后一个元素
c.data();//返回array的起点地址
qsort(c.data(), msize, sizeof(long), compareLong); // 快速排序，指定地址，多少个元素，每个元素大小是多少，比较方法是怎样
bsreach(&target, c.data(), msize, sizeof(long), compareLong); //二分查找，指定查找对象

vector

动态增长数组。容器容量按2倍增长，扩充的时候需要找到和当前容量两倍大的连续的内存才行。
有三个迭代器，所以sizeof(vector)的大小为12。在进行增长的时候，会调用大量的构造函数。每次Push.back申请的空间以两倍增长，会申请预留空间；size是大小，capacity是申请的内存空间大小；不可以原地扩充，都是重新找一个两倍大的地方；

vector c;
c.push_back(3);
c.size();
c.capacity();
c.front();
c.back();
c.data();//起始地址
auto it = ::find(c.begin(), c.end(), target);//顺序查找

list

双向链表

list c;
c.max_ize();
c.size();
c.front();
c.back();
c.push_back(2);
c.sort();//list容器中自己的sort，调用的是标准库种的sort，这个和容器特性有关，那么为什么要这么设计呢？因为list的内存分配不连续，而::sort()里面是针对连续空间实现的排序算法，因此::sort不实用list排序

forward_list

单向链表，只提供头插法，尾插效率太低，计算size效率太低，寻找back效率太低，不提供。

forward_list c;
c.push_front(); //头插
c.max_size();
c.front();
//无c.back()
//无c.size()

deque

类似vector，两边都能扩充，逻辑上是连续的，但是物理上是分段连续的。有一个map来存放这些段。当一个buffer使用完的之后，会有一个新的buffer，这个buffer的地址放在map中。map会将这些分段的node给串起来。node指向map的那一块，first和last指的是node的边界。cur指向当前。node的元素还会有一个start和finish的迭代器，保存双向队列的两端头元素。map不够大，也会二倍扩充。

deque c;
c.push_back(2);
c.pop_back();
c.push_front(2);
c.pop_front();
c.max_size();
c.front();
c.back();

stack和queue没有自己的数据结构，是借用deque，是容器适配器，不提供迭代器，不提供遍历操作。

stack

stack c;
c.push(2);
c.pop();
c.top();
c.size();

queue

queue c;
c.push(2);
c.pop();
c.front();
c.back();
c.size();

multiset

multiset c;
c.insert(2);//会放在该放置的位置，放进去就会被排序
c.begin();
c.end();
c.size();
c.max_size();
c.find(target);//内置find，红黑树

multimap

通过key来排序，查找

multimap c;
c.insert(pair(2, 3));//会放在该放置的位置，放进去就会根据key被排序
c.begin();
c.end();
c.size();
c.max_size();
c.find(target);//内置find，红黑树
//无c.front()
//multimap不能用下标[]来插入值

set&&map

这两种方法和前面的multi**使用方法一样，只是不允许有重复的key值。
set和map大多数工作都是交给RBTree，从这个角度看，set和map也是一种容器适配器

HashTable

使用hash表，冲突采用拉链法。使用hashtable最困难的事决定使用什么hash函数。为防止其中一个链表过长，影响效率；设定为当元素数量>篮子数量时，将其进行打散；将篮子增加两倍，重新进行计算；不是严格的两倍，选择素数，两倍数量附近的一个素数作为新的篮子数量；
